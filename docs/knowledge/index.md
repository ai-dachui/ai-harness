# Knowledge Index

Evergreen topic documentation, organized by domain.

## Structure

Each file in this directory covers one topic with four sections:
1. **What it is** — definition and scope
2. **Core concepts** — key ideas and mental models
3. **How to use** — practical application
4. **Sources & traceability** — where this knowledge came from

## Categories

Organize your knowledge files into categories that make sense for your domain.

Example categories:
- `frameworks/` — Framework-specific knowledge
- `infrastructure/` — Infrastructure and deployment
- `tools/` — Tool usage patterns
- `processes/` — Team processes and workflows
- `domain/` — Business domain knowledge

## Adding Knowledge

Knowledge typically "graduates" from other sources:
- A pattern appears 2+ times in retro → becomes a knowledge entry
- A research report has reusable conclusions → key points graduate here
- A runbook's "why" section grows complex → extract into knowledge
